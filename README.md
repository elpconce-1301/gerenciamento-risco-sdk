# gerenciamento-risco-sdk
Artefatos de gestão de riscos e comunicação do projeto real Violeta SDK.

<h1 style="color:#6A0DAD; font-family:Segoe UI, Arial; font-size:32px;">
📘 Projeto Violeta SDK — Gestão de Riscos e Comunicação
</h1>

<p style="font-size:15px;">
Este repositório contém os artefatos desenvolvidos para a atividade prática da pós-graduação em Inteligência Artificial, 
relacionados à gestão de riscos e comunicação em projetos de software.
O cenário utilizado é um <strong>projeto real</strong>: a integração do <strong>SDK Violeta</strong>, escrito em Python, 
com o catálogo corporativo de APIs, utilizando modelos de linguagem (LLMs) para auxiliar desenvolvedores diretamente na IDE.
</p>

---

<h2 style="color:#4B0082; font-family:Segoe UI, Arial;">
🏗️ Cenário do Projeto
</h2>

<p>
O objetivo do projeto é permitir que o desenvolvedor consulte, dentro da própria IDE, quais APIs estão disponíveis para determinada 
funcionalidade, sem precisar acessar portais externos. O SDK interpreta a intenção do dev, consulta uma base interna sincronizada 
com o catálogo de APIs e retorna sugestões inteligentes.
</p>

<h3 style="color:#4B0082; font-family:Segoe UI, Arial;">Nome do projeto</h3>
<p><strong>Integração do SDK Violeta com o Catálogo de APIs corporativo usando LLMs.</strong></p>

<h3 style="color:#4B0082; font-family:Segoe UI, Arial;">Objetivo</h3>
<p>
Facilitar o trabalho dos desenvolvedores, permitindo que consultem APIs disponíveis diretamente da IDE, por meio de uma biblioteca 
Python que utiliza LLMs para interpretar intenções e sugerir endpoints adequados.
</p>

<h3 style="color:#4B0082; font-family:Segoe UI, Arial;">Escopo resumido</h3>

<ul>
  <li>Biblioteca Python (SDK) que recebe uma descrição da funcionalidade desejada.</li>
  <li>Consulta ao catálogo de APIs corporativo por meio de um serviço interno.</li>
  <li>Uso de LLMs para interpretar a intenção e sugerir APIs relevantes.</li>
  <li>Integração com IDEs como VSCode ou PyCharm.</li>
  <li>Autenticação via serviços internos (ms-agents-registry, ms-auth-api, OIM).</li>
</ul>

<h3 style="color:#4B0082; font-family:Segoe UI, Arial;">Stakeholders</h3>

<ul>
  <li>Desenvolvedores internos</li>
  <li>Equipe de Arquitetura</li>
  <li>Time de Segurança e Compliance</li>
  <li>Time de IA/ML</li>
  <li>Time responsável pelo catálogo de APIs</li>
</ul>

<h3 style="color:#4B0082; font-family:Segoe UI, Arial;">Tecnologias envolvidas</h3>

<ul>
  <li>Python</li>
  <li>LLMs (Azure OpenAI ou equivalente)</li>
  <li>Catálogo corporativo de APIs</li>
  <li>AKS (Azure Kubernetes Service)</li>
  <li>Serviços internos de autenticação (OIM, ms-auth-api)</li>
</ul>
